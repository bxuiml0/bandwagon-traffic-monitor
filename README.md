## 搬瓦工流量是什么？先说清楚基本概念

每台搬瓦工 VPS 都有一个月流量上限，不同套餐不一样：入门款通常是 500GB 或 1TB/月，高配套餐可以到 2TB 甚至更高。这个流量是双向计算的——你往外传数据、别人往你服务器传数据，都算进去。

流量重置不按自然月来，而是按你**购买当天**的日期来算周期。比如你是 5 号买的，那每个月 5 号流量清零、重新计算。不少人以为是每月 1 号重置，等了大半个月发现不对，就是因为这个。

## 搬瓦工查看流量的主要方式

### 方式一：KiwiVM 控制面板（最直接）

KiwiVM 是搬瓦工自研的管理后台，查看流量用它最快。步骤如下：

1. 打开搬瓦工官网，用账号密码登录
2. 点击顶部导航 **Services → My Services**，找到你的 VPS
3. 点击右侧的 **Open KiwiVM** 按钮，进入该台 VPS 的控制面板
4. 进入 KiwiVM 首页后，找到 **Bandwidth usage** 这一栏
5. 这里会显示三个关键数据：**已用流量 / 总流量**（格式类似 `25 / 1000 GB`）、以及下方的 **Resets**（即流量重置日期）

数据是实时更新的，刷新页面就能看到最新状态。

不用一直盯着面板，搬瓦工会在你用到 **80%、90%、95%** 时分别发邮件提醒。很多人流量超了才发现是因为这些邮件进了垃圾箱，建议先把搬瓦工发件邮箱加进白名单。

### 方式二：KiwiVM API 查询（适合经常要查的人）

如果你管着好几台 VPS，每次都进 KiwiVM 面板翻来翻去挺烦的。KiwiVM 提供了 API 接口，可以直接请求一个链接查到流量数据。

找 API 信息的位置：在 KiwiVM 面板左侧菜单找到 **API** 选项，里面有你的 **VEID**（VPS 编号）和 **API Key**（私钥，别泄漏）。

拿到这两个值之后，自己拼一个查询请求发过去，返回结果里包含：总流量、已用流量、剩余流量、流量重置日期（北京时间），以及到重置日还剩几天。

适合用脚本做监控或者整合进自己的运维工具里。偶尔查一次还是直接开 KiwiVM 更省事。

---

### 方式三：KiwiVM 的 Detailed statistics（流量历史图）

想知道流量是哪段时间用掉的，可以在 KiwiVM 左侧菜单点 **Detailed statistics**。里面有三张图：网络流量（Network I/O）、CPU 使用率、硬盘读写速度，还能切换按天 / 周 / 月 / 年的统计周期。

有时候流量用得特别快但说不出哪里有问题，拉一下这里的历史曲线，通常能找到异常消耗的时间点。

## 流量重置日到底是哪天？

进 KiwiVM 面板，Bandwidth usage 下方有个 **Resets** 字段，显示的就是下次流量重置日期。

有几个细节要注意：

- 时区换算：搬瓦工显示的日期是美国时间，换算到北京时间大概会有 1 天左右误差，别卡着点等
- 机房迁移会消耗流量：迁机房的时候数据传输本身会计入总流量，迁之前最好确认剩余流量够用
- 部分套餐进入 CN2 机房后流量变三分之一：这是老 KVM PROMO 套餐的特殊规定，如果你发现流量莫名其妙少了很多，可以去 KiwiVM 看看当前机房是不是这种情况

## 流量用完了怎么办？"Data transfer limit exceeded" 的处理方法

流量耗尽之后，VPS 会被直接暂停，KiwiVM 面板会显示：

> This service is currently suspended. There is 1 outstanding issue:  
> Reason: Data transfer limit exceeded  
> How to resolve: Data transfer counter resets automatically every month.

看到这个不用慌。有两条路：

**等重置（适合不急的情况）**：流量每月在购买周年日自动清零，重置之后 VPS 自动恢复，不需要手动操作。

**升级套餐（适合等不及的情况）**：搬瓦工不支持单独购买额外流量，只能整体升级到更高配置的套餐。升级之后流量限额随之提升，VPS 立刻恢复可用。

## 几个容易被忽视的流量"漏洞"

我自己用搬瓦工这段时间，碰到过几次流量莫名其妙消耗特别快的情况，后来排查下来主要是这几个原因：

**1. 私有网络没开**：搬瓦工同一机房内可以用私有 IP 建立内网，内网传输不占公共流量。如果你有两台机器需要频繁传数据，没开这个功能的话流量消耗会成倍增加。

**2. 服务暴露在公网被扫**：端口开着又没配访问限制，各路扫描爬虫会一直往你服务器发包，全算进流量里。用 iptables 或者 fail2ban 做一下基本防护，能省不少。

**3. 切机房的数据迁移**：换机房的时候系统会把磁盘数据整个搬一遍，占用的流量是实打实的。大硬盘套餐迁机房前最好确认一下当月剩余流量够不够。

## 常见问题

**Q：搬瓦工流量是单向计费还是双向计费？**  
双向。上传和下载都会计入月流量，不要以为只有往外传数据才算。

**Q：为什么套餐写的 1TB，实际用到 DC3 CN2 机房流量变少了？**  
部分老 KVM 套餐迁入 CN2 机房后流量会变为三分之一，这是搬瓦工的套餐规定。买之前看清楚套餐说明，CN2 GIA-E 系列没有这个限制。

**Q：流量用完之后，能提前恢复吗？**  
搬瓦工 KiwiVM 面板有一个"临时恢复"功能，可以短暂解除暂停状态，但只是临时的，治标不治本。真正解决方法要么等重置日，要么升级套餐。

**Q：流量快用完了会有提醒吗？**  
会。用到 80%、90%、95% 时分别有邮件通知。确保收件邮箱正常，别被过滤掉。

**Q：有没有办法买额外的流量包？**  
不支持。搬瓦工没有单独的流量附加包，只能升级到更高配置的套餐。这一点确实不够灵活，算是用搬瓦工要接受的限制之一。
